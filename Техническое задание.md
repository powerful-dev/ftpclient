# **Техническое задание на разработку FTP-клиента**

## **Общее описание**

Разработать FTP-клиент с двухпанельным интерфейсом в тёмной теме с современным дизайном.

## **Структура интерфейса**

1. **Верхняя панель**: Вкладки подключений для быстрого переключения между различными FTP-серверами  
2. **Средняя панель**: Две области просмотра каталогов (левая и правая)  
   * Над каждой панелью отображается строка текущего пути  
   * В строке текущего пути открывается история открытых директорий  
   * Каждая панель содержит список файлов и каталогов текущего уровня  
3. **Нижняя панель**: Строка состояния, отображающая прогресс текущих операций и статус соединения  
4. **Боковая панель:** Слева панель со списком сохранённых серверов, к которым можно подключиться

## **Функциональные требования**

### **Управление подключениями**

* Создание, редактирование и удаление профилей подключений  
* Отображение активных подключений во вкладках с возможностью закрытия, дублирования, закрытия всех владок  
* Сохранение настроек подключений (адрес, порт, логин, пароль, режим соединения)  
* Сохранение последнего открытого каталога слева и справа (открывать снова при подключении) и истории каталогов слева и справа для каждого подключения

### **Просмотр файловой системы**

* Обновление содержимого каталогов вручную и автоматически (при открытии каталога и каждые 10 секунд)  
* Сортировка по имени, размеру, дате изменения  
* Фильтрация файлов по маске или типу (поиск файлов в текущей папке, поиск везде)

### **Контекстное меню для файлов и каталогов**

При нажатии правой кнопкой мыши на файле или каталоге должно появляться меню со следующими опциями:

* Скачать в открытую папку / Загрузить в открытую папку  
* Открыть в стандартном приложении (для локальных файлов)  
* Переименовать  
* Переместить (куда)  
* Создать копию (куда)  
* Изменить права доступа (chmod)  
* Просмотреть свойства  
* Удалить  
* Создать новый каталог  
* Создать новый файл

### **Передача файлов**

* Перетаскивание файлов и папок между панелями (drag and drop)  
* Копирование и перемещение файлов и каталогов  
* Список передач с отображением прогресса по каждой задаче  
* Возможность приостановки и возобновления передачи

### **Профили подключения**

* Список сохранённых профилей (подключений к серверам) с сортировкой по дате добавления  
* Возможность удалить и добавить профиля  
* При новом подключении указываются:  
  * Адрес сервера  
  * (Порт при желании)  
  * Протокол подключения  
  * Логин (при желании)  
  * Пароль или SSH-ключ (при желании)  
  * Сохранить ли профиль (по умолчанию да).   
    Если да, то дополнительное обязательное поле  
    * Название профиля  
    * Цвет профиля (разноцветные шарики). По умолчанию случаный цвет

## **Технические требования**

* Поддержка протоколов FTP, FTPS, SFTP, Amazon S3. Используй thephpleague/flysystem ([https://flysystem.thephpleague.com/docs/](https://flysystem.thephpleague.com/docs/)) для подключений и работы с файлами. Если для SSH нужно будет, можно использовать дополнительную библиотеку, например, phpseclib  
* Возможность работы с несколькими соединениями одновременно  
* Учет и отображение прав доступа к файлам  
* Кроссплатформенность (Windows, macOS, Linux)  
* Для копирования и перемещения использовать команды SSH для SFTP, иначе, если небольшой размер, то временное хранилище не диске

## **Элементы интерфейса**

* Строка пути должна иметь функцию навигации "хлебные крошки"  
* Подсветка текущего активного элемента тонкой линией акцентного цвета  
* Эффекты при наведении курсора: легкое изменение цвета фона элемента

## **Отзывчивость интерфейса**

* Быстрый отклик интерфейса при навигации  
* Асинхронная загрузка списков файлов без блокировки интерфейса  
* Асинхронная передача файлов без блокировки интерфейса  
* Автоматическое переподключение при разрыве соединения. При неудаче, повтор каждые 5 секунд или по нажатию кнопки “попробовать снова”

## **Дополнительные возможности**

* Многоязычный интерфейс  
* Закладки для часто используемых каталогов  
* Быстрая передача файлов (только для SFTP, так как есть доступ к консоли). При передаче больше 10 файлов или размера больше 50 МБ архивировать все файлы, которые нужно передать, архиватором, который поддерживает сервер, загружать на сервер, а затем распаковывать. (Решает проблему, когда много мелких файлов передаются медленно)  
* Импорт профилей подключения из FileZilla и WinSCP

## **Требования к безопасности**

* Шифрование соединений (для FTPS и SFTP)  
* Безопасное хранение учетных данных. (Это важно продумать, чтобы другие приложения не могли получать доступ. При необходимости можно шифровать или использовать какую-то защиту из PHPNative)  
* Возможность работы как с паролями, так и с ключами SSH для SFTP
